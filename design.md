# 🧭 New Portfolio — 웹사이트 기획서
> "읽는 포트폴리오가 아니라, 경험하는 포트폴리오"  
> 풀스택 개발자 기준 | Minimal + Seamless + Visually Unforgettable

---

## 🎯 핵심 방향

기존 PortfoliOh의 다음 단계.  
기능을 쑤셔넣는 게 아니라, **하나의 흐름으로 방문자를 이끄는 구조**.  
인터랙션 자체가 Product Engineer로서의 실력을 증명한다.

| 기존 PortfoliOh | New Portfolio |
|---|---|
| 기능 중심 맥시멀리스트 | 경험 중심 미니멀리스트 |
| 기억에 안 남음 | 스토리로 기억됨 |
| 기술 스택 나열 | 문제 → 해결 → 결과 |

## 🎨 시각적 컨셉: "Minimalism with Depth"

**Aesthetic:** Apple 제품 페이지 같은 미니멀리즘 + 고급스러운 인터랙션  
**Theme:** Glassmorphism(투명도) + Bento Grid(정교한 그리드 레이아웃)  
**Key Animation:** 스크롤에 반응하는 Progressive Content Reveal

### 컬러 시스템

흑백 베이스 + 포인트 컬러 1개

- 배경: `#0A0A0A` (딥 블랙) 또는 `#FAFAFA` (오프 화이트)
- 포인트: `#7B2D3E` (와인 레드) — 와인바 스토리와 시각적 연결
- 포인트 사용처: 호버 강조 / 현재 섹션 인디케이터 / CTA 버튼

### 타이포그래피

대형 sans-serif가 메인 비주얼 역할. 이미지를 대체한다.  
모든 텍스트는 sentence case. ALL CAPS / 과도한 볼드 금지.

### 애니메이션 원칙

- 모든 모션은 목적이 있어야 한다 — 장식용 금지
- CSS transition 위주, 필요할 때만 JS
- 무겁지 않게 — 200ms 이하 또는 스크롤 연동

## 섹션별 구현 아이디어

---

### 🌌 Section 01 — Hero (Identity)

**컨셉: "공간이 말을 걸다"**

브라우저를 열면 전체 화면에 딱 한 줄만 등장한다.  
화면 중앙에 아주 큰 타이포그래피. Hero 이미지 없음.

**인터랙션:**
- 사용자가 화면을 클릭하면 문구가 순서대로 바뀜:
  ```
  Developer → Designer → Wine Bar Owner → Product Engineer (마지막에 고정)
  ```
- 마우스 커서를 따라다니는 부드러운 조명(spotlight) 효과
- 텍스트가 한 단어씩 타이핑되거나, 스크롤하면 문장이 분해·재조합되는 모프 애니메이션
- 마우스 움직임에 따라 텍스트가 아주 미세하게 기울어지는 3D tilt parallax

**핵심 문구:**
> "나는 코드를 짜는 사람이 아니라, 경험을 빚는 사람입니다."

첫 3초 안에 "이 포트폴리오는 다르다"는 느낌을 줘야 한다.

### 🍷 Section 02 — Bridge (Storytelling)

**컨셉: "오프라인 감각이 디지털로 전이되는 순간"**

일반적인 타임라인 대신, 오프라인과 디지털의 연결을 시각화.

**비주얼 구성:**
- 좌측: 와인바 공간 사진
- 우측: 코드 터미널

**핵심 애니메이션:**
> 스크롤을 내리면 와인 잔의 액체가 쏟아지며 **데이터 라인(Line)으로 변해** 터미널로 흘러 들어가는 시각 효과

**수평 스크롤 타임라인 (정거장 구조):**

1. 🍷 와인바 운영 — "오프라인에서 사람의 선택을 관찰했다"
2. 📱 Flutter 부트캠프 — "처음으로 서비스라는 개념을 이해했다"
3. 🎨 UX/UI 부트캠프 — "기능이 아니라 경험을 설계하는 사람이 됐다"
4. 🤖 AI 풀스택 과정 — "이제 아이디어를 끝까지 구현할 수 있게 됐다"

배경색이 스크롤에 따라 서서히 변함: 웜톤(와인바) → 뉴트럴 → 쿨톤(테크)

### 🍱 Section 03 — Projects (The Proof)

**컨셉: "카드가 아니라 케이스 스터디"**

프로젝트를 나열하지 않는다.  
크기가 다른 사각형들이 모인 **Bento Grid** 형태로 배치.

**케이스 스터디 구조 (각 프로젝트 공통):**
```
문제 — 왜 만들었나, 어떤 불편함에서 출발했나
해결 — 어떤 방식으로 접근했나, 핵심 결정은 무엇이었나
결과 — 실제로 어떻게 작동하는가, 무엇을 배웠나
```

**프로젝트별 유니크 인터랙션:**

| 프로젝트 | 인터랙션 아이디어 |
|---|---|
| **Uncorked** (Hero) | 마우스를 올리면 와인 향기가 퍼지는 듯한 블러(Blur) 애니메이션 |
| **EZAir** (Hero) | 카드 내부에서 비행기가 구름 사이를 뚫고 지나가는 미니멀한 SVG 애니메이션 |
| **PortfoliOh** (서브) | 하이라이터 기능 자체를 카드에서 체험 가능하게 |
| **Dev Quest** (서브) | 현재 진행형을 강조, 로딩 애니메이션으로 "개발 중" 표현 |
| **Nomad Market / Onjung** (보조) | 작은 카드로, "초기 성장 과정"으로 배치 |

**공통 인터랙션:**
- 프로젝트 카드 클릭 시 화면을 꽉 채우는 케이스 스터디로 확장
- hover 시 실제 사이트 미리보기가 카드 내부에서 재생
- 각 프로젝트마다 "이 프로젝트에서 가장 어려웠던 결정" 한 줄 포함 → 사고방식 드러냄

### ⚙️ Section 04 — Skills (The Layer)

**컨셉: "도구가 아니라 레이어"**

기술 스택을 로고 나열로 보여주지 않는다.  
실제 서비스가 어떻게 구성되는지 **단면도처럼 시각화**.

```
┌─────────────────────────────────────┐
│  🤖 AI / Product Layer              │  AI API 연동, 추천 기능, 챗봇
├─────────────────────────────────────┤
│  ⚙️ Backend Layer                   │  Python(FastAPI), Java(Spring Boot)
├─────────────────────────────────────┤
│  🎨 Frontend Layer                  │  HTML/CSS/JS, React(학습중), UX 설계
└─────────────────────────────────────┘
```

- 각 레이어 클릭 시 → 기술 + 역할 설명이 슬라이드인으로 등장
- 학습 중인 기술은 점선 레이어로 표시 (진행 중임을 시각적으로 표현)
- 기술 이름만 나열하지 않음 → 반드시 "무엇을 위해 쓰는가" 맥락과 함께

> 핵심 메시지: **"기술을 사용하는 것이 아니라, 문제 해결을 위해 선택한다"**

### 🤖 Section 05 — AI Co-Pilot (Seamless Integration)

**컨셉: "포트폴리오를 이해하는 AI 가이드"**

이전 PortfoliOh의 챗봇보다 발전된 형태.

**작동 방식:**
- 화면 우측 하단에 숨어있다가 사용자가 특정 단어 (예: "협업", "백엔드")를 드래그하면 즉시 관련 프로젝트나 생각을 팝업으로 띄워줌
- 단순한 대화창이 아니라, 포트폴리오 전체 데이터를 알고 있는 '가이드' 역할

**기술 스택:**
- OpenAI API + 벡터 데이터베이스 (RAG)
- "나의 철학"을 학습한 AI 모델 연결
- Python(FastAPI)으로 가벼운 API 서버 구축

### 👤 Section 06 — About

**컨셉: "스크롤로 완성되는 자기소개"**

스크롤을 내릴수록 엘리베이터 피치가 한 문장씩 fade-in으로 등장하며 완성된다.

```
1. "오프라인 공간에서 고객 경험을 설계하는 일을 해왔습니다."
2. "좋은 경험은 물리적인 공간뿐 아니라, 디지털에서도 설계될 수 있다는 걸 깨달았습니다."
3. "지금은 사용자 경험을 중심으로 웹 서비스를 설계하고 구현하고 있습니다."
4. "프론트엔드부터 백엔드까지, 아이디어를 실제 제품으로 끝까지 만듭니다."
5. "목표는 더 많은 사람이 실제로 사용하는 서비스를 만드는 것입니다."
```

마지막 문장 등장 후 CTA 버튼 등장: **"같이 만들어볼까요?"**

## 🌟 신선함을 주는 추가 아이디어 (Anti-Boring)

### Dark / Light Mode 전환 (심화)
단순히 색만 바뀌는 게 아니라, 테마를 바꾸면 분위기 자체가 바뀜
- Light Mode → Wine Bar 컨셉 (따뜻한 웜톤, 오프라인 감성)
- Dark Mode → Dev Lab 컨셉 (차가운 쿨톤, 테크 감성)

### Sound Design
아주 미세한 효과음 추가:
- 버튼 클릭 시 '틱'
- 섹션 전환 시 부드러운 '슝'

### Terminal Easter Egg
백엔드 프로젝트 섹션에 실제로 작동하는 미니 터미널 창.  
`help` 명령어를 치면 숨겨진 이스터 에그 등장.

### Highlighter 2.0 (PortfoliOh 계승)
사용자가 하이라이트한 내용을 DB에 저장하고,  
마지막에 **"당신이 주목한 나의 역량 리포트"**를 이미지/PDF로 생성.

## 🛠 기술 스택 (구현)

| 역할 | 기술 | 이유 |
|---|---|---|
| 마크업 | HTML5 | 시맨틱 구조 |
| 스타일 | CSS3 + CSS Variables | 테마 관리, 애니메이션 |
| 인터랙션 | Vanilla JS | 외부 의존성 최소화 |
| 스크롤 애니메이션 | Intersection Observer API | 라이브러리 없이 구현 |
| 수평 스크롤 | CSS scroll-snap | 부드러운 네이티브 스크롤 |
| AI 백엔드 | Python (FastAPI) | AI API 연결, 챗봇 서버 |
| 배포 | Vercel (FE) + Render (BE) | 빠른 배포, 무료 |

> React는 현재 학습 중이므로 Vanilla JS로 구현.  
> "라이브러리 없이도 이런 인터랙션을 만든다"는 것 자체가 실력 증명.
